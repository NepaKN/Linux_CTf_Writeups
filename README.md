# Linux_CTF_Writeups
This repo is about how I completed progressive Capture The Flag Challenges. Testing the Linux command line skills. Here, I am sharing the learning experience not the solutions. You definitely have distinct experience even you walk the same path. Experience the way as you go.

LEARN TO CLOUD CTF Challenges
├── ⭐ Beginner (2 challenges)
│   ├── 1. The Hidden File – Find and read a hidden file in ctf_challenges 
│   └── 2. The Secret File – Locate a file containing "secret" in its name under your home directory
├── ⭐⭐ Intermediate (13 challenges)
│   ├── 3. The Largest Log – Find and read an unusually large file in /var/log
│   ├── 4. The User Detective – A user with UID 1002 has a flag in their login configuration
│   ├── 5. The Permissive File – Find a suspicious file with wide-open permissions under /opt
│   ├── 6. The Hidden Service – Something is listening on port 8080. Connect to it
│   ├── 7. The Encoded Secret – Find and decode an encoded flag in ctf_challenges
│   ├── 8. SSH Key Authentication – Configure SSH key authentication and find a hidden flag
│   ├── 9. DNS Troubleshooting – Someone modified a critical DNS config file. Fix it
│   ├── 10. Remote Upload – Transfer any file to ctf_challenges to trigger the flag
│   ├── 11. Web Configuration – The web server is running on a non-standard port. Find and fix it
│   ├── 13. Cron Job Hunter – A scheduled task contains a hidden flag. Find and read it
│   ├── 15. Archive Archaeologist – A flag is buried inside nested archives. Dig it out
│   ├── 16. Symbolic Sleuth – Follow the trail of symbolic links to find the flag
│   └── 17. History Mystery – Someone typed a flag in their command history. Find it
└── ⭐⭐⭐ Advanced (3 challenges)
    ├── 12. Network Traffic Analysis – Someone is sending secret messages via ping packets
    ├── 14. Process Environment – A running process has a secret in its environment. Extract it
    └── 18. Disk Detective – A flag is hidden in filesystem metadata. Investigate mounted filesystems
